The Opportunity

The global AI security market is projected to reach $60.6B by 2028 (MarketsandMarkets, 2024), growing at 23% CAGR. Yet every major player — CrowdStrike, Palo Alto, Darktrace — treats security as a purely technical problem. They miss the critical regulatory and legal dimension that is now reshaping enterprise AI governance in Canada, the EU, and beyond.

VIGILAX is the only AI security engine that integrates constitutional rights compliance as a mandatory enforcement gate. In the Canadian regulatory context — where the Privacy Commissioner, CCTS, CRTC, and Charter challenges are intensifying — this is not a differentiator. It is table stakes for enterprise customers in healthcare, telecom, financial services, and government.

Francisco Holdings Inc. is uniquely positioned to own this intersection. The founder brings 20+ years of Canadian constitutional advocacy, deep telecom regulatory experience, and direct domain expertise in AI abuse patterns, consumer rights enforcement, and evidence law.

The Product

VIGILAX is a four-layer proprietary security intelligence engine deployed as a multi-tenant SaaS platform:

  • Perception Layer: Real-time ingestion of network traffic, endpoint telemetry, behavioral biometrics. Native API connectors for AWS CloudWatch, Azure Sentinel, GCP Security Command Center, Splunk, QRadar, and pfSense. Natural language threat intelligence feed parsing.
  • Cognition Layer (Claude API): Eight specialized AI agents operating on a three-tier model strategy (Opus 4.8 / Sonnet 4.6 / Haiku 4.5). The Rights-Preservation Protocol is the patentable core: every threat response is evaluated against Charter s.7, PIPEDA, PHIPA, and evidence admissibility rules before execution. No other product does this.
  • Response Layer: Automated countermeasures for pre-cleared threats. Human-in-the-loop escalation for rights-sensitive actions. Full audit trail generation for CCTS, CRTC, FCAC, Ontario Small Claims, and court proceedings.
  • Memory Layer: Persistent threat intelligence graph across all 45+ subsidiary companies. Cross-company attack pattern recognition. Federated learning — every incident makes the entire network smarter.
Intellectual Property
  • VIGILAX-IP-001 — Constitutional Rights Gate: Method wherein every AI enforcement action is submitted to a constitutionally-aware sub-agent (ARBITER) before execution, with automatic blocking of Charter/PIPEDA-violating actions. No prior art exists for this architecture.
  • VIGILAX-IP-002 — LLM Evidence Sealing: System for automatically processing participant-recorded material (s.184(2)(a) Criminal Code), extracting structured claims via LLM, and sealing output with SHA-256 + RFC 3161 timestamp into a write-once Postgres table — producing court-admissible evidence without human transcription.
  • VIGILAX-IP-003 — Three-Tier LLM Routing: Routing architecture assigning security tasks across Opus/Sonnet/Haiku based on severity, latency budget, and constitutional sensitivity. Maintains sub-100ms for real-time classification while preserving deep reasoning for high-stakes decisions.
  • VIGILAX-IP-004 — Telecom Abuse Detection: Constitutional AI module detecting unauthorized billing, false notice claims, partial-refund obstruction — automatically generating regulatory filing documents (CCTS/CRTC/FCAC).
  • VIGILAX-IP-005 — Role-Locked Multi-Agent Mesh: Architecture where each agent's system prompt contains a role declaration that a guardian agent verifies on every inter-agent message — preventing prompt injection propagation across agent boundaries.
